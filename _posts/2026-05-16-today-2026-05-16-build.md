## §5. 시행착오 (Lessons Learned)

> 본 vault 의 *불완전* 시도·실패·우회. 미래 자신과 다른 시도자에게 *지름길*.

### 사례 #7 — Notion 외부 공유 한계 (회상)
- **문제**: Notion 페이지를 *완전 공개 + 검색 가능* 으로 만들기 어렵고, 본인 워크스페이스 누적 자산이 *섞임*
- **결론 → D-3**: vault wiki/ 가 *공유 가능 형태* 의 별도 채널. Notion 은 *transient capture* 만 유지
- **영구 기록**: vault 가 *공유 채널의 source of truth*, Notion 은 *인박스*

### 사례 #8 — MCP 등록 채널 혼동 (`settings.json` vs `claude.json`)
- **문제 (2026-05-15)**: `~/.claude/settings.json` 의 `mcpServers` 직접 편집 → `claude mcp list` 미인식
- **원인**: 진짜 store 는 `~/.claude.json` (Claude Code CLI). `claude mcp add` 명령 *유일* 정석
- **해결**: `claude mcp add <name> [-s user|project] [-e KEY=VAL] [--transport http <url>] -- <command>` 패턴
- **영속화**: global `~/.claude/CLAUDE.md` 의 *🔧 MCP 등록 전용 체크리스트* 섹션 + `tool-setup-guide` 스킬 §8 영구 기록
- **교훈**: *결과 ≠ 작동 메커니즘*. 기존 형식 작동 가정 X, 공식 CLI 명령 사용 + `claude mcp list` 검증 의무

### 사례 #9 — 회의록을 AI 에게 시키면 *지맘대로*
- **문제**: 클로드 코드에 회의록 정리 시키면 매번 다른 구조
- **원인**: 필터 6 항목 명시 안 함 (= CIF 의 F 부재)
- **해결**: 회의록 형식 = ① 핵심 요약 ② 결정사항 ③ 액션 리스트 ④ 액션 담당자 ⑤ 주요 인사이트 ⑥ 미팅 로그
- **영속화**: 본 회의록 + [[2026-05-14-think-subscription-live]] 이 *형식 적용 1·2 호*. 모든 회의록 표준 (D-6)
- **교훈**: AI 자유도 ≠ 좋음. *필터 명시* 가 결과의 80%.

### 사례 #10 (보너스) — *2 주 만들고 버린* 함정 회피
